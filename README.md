#### 关系的完善
用户的关系是个人行为，并非是双方的行为，所以将原本的```each```关系拆为4份，分别表示双方的关系

#### *为什么放弃*
很简单在做完```关系的完善```部分后，效率尿崩，而且可能是因为节点和关系的数量不断增加，即便不更改关系效率依然很慢。如图~
